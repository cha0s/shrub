Package overview

Packages are how Shrub organizes functionality. Packages may be provided for the server or the client (or both).

This page provides a listing of packages in this project, along with a short description of the functionality they provide.

Client-side

  • shrub-angular

    Hang up any sandbox if we don't need it anymore.

  • shrub-core

    Core functionality.

  • shrub-example

    Define some routes, set a nav, show some stuff off!

  • shrub-form

    Define a directive for Angular forms, and a service to cache and look them up later.

  • shrub-limiter

    Define a TransmittableError for the limiter.

  • shrub-rpc

    Define an Angular service to issue remote procedure calls.

  • shrub-schema

    Provide the JugglingDB schema as an Angular service.

  • shrub-socket

    Provide an Angular service wrapping Socket.IO.

  • shrub-ui

    Define user interface components.

  • shrub-user

    User operations, model, etc.

Server-side

  • shrub-angular

    A sandboxed version of Angular, for clients lacking JS.

  • shrub-assets

    Serve different JS based on whether the server is running in production mode.

  • shrub-config

    Client-side configuration.

  • shrub-core

    Implements various core functionality.

  • shrub-express

    An Express HTTP server implementation, with middleware for sessions, routing, logging, etc.

  • shrub-form

    Handle form and method parsing, and submission of POST'ed data into the Angular sandbox.

  • shrub-http

    Manage HTTP connections.

  • shrub-limiter

    Limits the rate at which clients can do certain operations, like call RPC endpoints.

  • shrub-nodemailer

    Renders and sends email.

  • shrub-repl

    Runs a REPL and allows packages to add values to its context.

  • shrub-rpc

    Framework for communication between client and server through RPC

  • shrub-schema

    Tools for working with the database schema.

  • shrub-session

    Various means for dealing with sessions.

  • shrub-socket

    Manage socket connections.

  • shrub-socket.io

    SocketManager implementation using Socket.IO.

  • shrub-user

    User oprations.

  • shrub-villiany

    Watch for and punish bad behavior.
