Hook overview

Shrub implements message passing between packages through a hook system. Hooks may be invoked with pkgman.invoke(), and are implemented in packages by prefixing a $ to the hook name.

For instance, if we are implementing a package and want to implement the httpListening hook, our code would look like:

exports.$httpListening = ->

    # Your code goes here...

The list below was dynamically generated from the source code. There is a description and a list of implementing packages for each hook.

server.coffee

  • initialize

    Invoked when the server is just starting. Implementations should return a promise. When all returned promises are fulfilled, initialization continues.

    • angular (server)

    • express/errors (server)

  • initialized

    Invoked after the server is initialized.

server/AbstractHttp.coffee

  • httpInitializer

    Invoked before the server is bound on the listening port.

  • httpListening

    Invoked once the server is listening, but before initialization finishes.

  • httpMiddleware

    Invoked every time an HTTP connection is established.

    • angular (server)

    • assets (server)

    • config (server)

    • core (server)

    • express/errors (server)

server/AbstractSocketFactory.coffee

  • socketAuthorizationMiddleware

    Invoked when a socket connection begins. Packages may throw an instance of AbstractSocketFactory.AuthorizationFailure to reject the socket connection as unauthorized.

    • core (server)

  • socketRequestMiddleware

    Invoked for every socket connection.

server/audit.coffee

  • auditKeys

    Allows a package to specify unique keys for this request, e.g. IP address, session ID, etc.

    • core (server)

server/config.coffee

  • settings

    Invoked when the server application is loading configuration. Allows packages to define their own default settings.

    • angular (server)

    • core (server)

    • express/errors (server)

server/packages/config/index.coffee

  • config

    Allows packages to specify configuration that will be sent to the client.

    • config (server)
