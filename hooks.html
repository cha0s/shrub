Hook overview

Shrub implements message passing between packages through a hook system. Hooks may be invoked with pkgman.invoke(), and are implemented in packages by prefixing a $ to the hook name.

For instance, if we are implementing a package and want to implement the httpListening hook, our code would look like:

exports.$httpListening = ->

    # Your code goes here...

The list below was dynamically generated from the source code. There is a description and a list of implementing packages for each hook.

client/app.coffee

  • appConfig

    Invoked when the Angular application is in the configuration phase.

    • core (client)

    • example/home (client)

  • appRun

    Invoked when the Angular application is run.

    • angular (client)

    • core (client)

    • example (client)

client/modules/errors.coffee

  • transmittableError

    Allows packages to specify transmittable errors. Implementations should return a subclass of TransmittableError.

    • limiter (client, server)

    • user/login (client, server)

client/modules/packages/core/index.coffee

  • route

    Allow packages to define routes in the Angular application.

    • example/about (client)

    • example/home (client)

    • user/forgot (client)

    • user/login (client)

    • user/logout (client)

    • user/register (client)

    • user/reset (client)

  • routeMock

    Allow packages to define routes in the Angular application which are only defined during test mode.

    • core (client)

    • user/login (client)

  • routeAlter

    Allow packages to alter defined routes.

  • routeControllerStart

    Allow packages to act before a new route controller is executed.

    • ui/title (client)

client/modules/schema-client.coffee

  • models

    Allows packages to create models in the database schema.

    • user (client, server)

    • villiany (server)

  • modelsAlter

    Allows packages to alter any models defined.

    • user (client, server)

    • form (client)

server.coffee

  • initialize

    Invoked when the server is just starting. Implementations should return a promise. When all returned promises are fulfilled, initialization continues.

    • angular (server)

    • express (server)

    • rpc (server)

    • user (server)

  • ready

    Invoked after the server is initialized and ready.

    • repl (server)

server/AbstractHttp.coffee

  • httpInitializing

    Invoked before the server is bound on the listening port.

    • schema (server)

    • socket/factory (server)

  • httpListening

    Invoked once the server is listening, but before initialization finishes. Implementations should return a promise. When all promises are fulfilled, initialization finishes.

    • nodemailer (server)

  • httpMiddleware

    Invoked every time an HTTP connection is established.

    • angular (server)

    • assets (server)

    • config (server)

    • core (server)

    • express/errors (server)

    • express/logger (server)

    • express/routes (server)

    • express/session (server)

    • express/static (server)

    • form (server)

    • socket/factory (server)

    • user (server)

server/AbstractSocketFactory.coffee

  • socketAuthorizationMiddleware

    • core (server)

    • session (server)

    • user (server)

  • socketConnectionMiddleware

    Invoked for every socket connection.

    • rpc (server)

    • session (server)

    • socket/factory (server)
