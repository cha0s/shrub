A route is defined like:

  • validators: Validators are run before the call is received. They are defined as middleware.

  • receiver: The receiver is called if none of the validators throw an error.

Example:

validators: [ (req, res, next) -> if req.badStuffHappened next new Error "YIKES!" else next() ] receiver: (req, fn) -> if req.badStuffHappened fn new Error "YIKES!" else

Anything you pass to the second parameter of fn will be passed back to the client. Keep this in mind.

fn null, message: "Everything went better than expected!"

Don't pass req directly, since it can be mutated by routes, and violate other routes' expectations.

routeReq = {} routeReq[key] = value for key, value of req routeReq.body = data # Send an error to the client. emitError = (error) -> fn error: errors.serialize error # Log an error without transmitting it. logError = (error) -> logger.error errors.stack error

Send an error to the client, but don't notify them of the real underlying issue.

Transmit the error as it is directly to the client.
